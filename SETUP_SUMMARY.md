# Site Setup Summary

This document provides a comprehensive overview of the GitHub Pages site setup for security walkthroughs.

## What Was Built

A complete, production-ready GitHub Pages site using the **Chirpy Jekyll theme** for documenting:
- TryHackMe challenges and rooms
- Hack The Box machines (retired only)
- Network laboratory configurations
- Security projects and tools

## Architecture

### Theme: Chirpy
- **Source:** [cotes2020/jekyll-theme-chirpy](https://github.com/cotes2020/jekyll-theme-chirpy)
- **Style:** Clean, responsive layout with sidebar navigation
- **Features:** Built-in categories, tags, archives, table of contents, dark/light mode

### Static Site Generator: Jekyll
- **Version:** Compatible with GitHub Pages
- **Markdown:** Kramdown with Rouge syntax highlighting
- **Gem:** `jekyll-theme-chirpy`

## How It Works

### 1. Content Creation
- Authors create markdown files in `_posts/` directory
- Files follow naming convention: `YYYY-MM-DD-title.md`
- Front matter specifies metadata (title, date, categories, tags)

### 2. GitHub Pages Build
- Push to `main` branch triggers GitHub Actions workflow (`.github/workflows/pages.yml`)
- Jekyll processes markdown files into HTML using the Chirpy theme
- Site is deployed to `https://cta0930.github.io`

### 3. Navigation
- Homepage displays recent posts
- Sidebar links to Categories, Tags, and Archives (auto-generated by `jekyll-archives`)
- Tags and categories enable topic-based filtering

## Key Features

### ✅ Responsive Design
- Works on desktop, tablet, and mobile
- Chirpy sidebar collapses on smaller screens

### ✅ Category & Tag Organization
- Categories and tags auto-generated via `jekyll-archives`
- Chirpy permalink format: `/categories/:name/` and `/tags/:name/`

### ✅ Code Highlighting
- Syntax highlighting via Rouge
- Line numbers enabled in code blocks
- Supports all major languages

### ✅ Table of Contents
- Automatic per-post TOC enabled in `_config.yml`
- Renders in the right sidebar on desktop

### ✅ Easy Content Addition
- Simple markdown format with YAML front matter
- Auto date-based permalink: `/posts/:title/`

## Configuration Details

### _config.yml Settings

```yaml

# Identity
```

## Content Guidelines

### Post Front Matter Template

```yaml
---
layout: post
title: "Platform - Challenge Name"
date: YYYY-MM-DD
categories: [TryHackMe]  # or [HackTheBox], [NetworkLabs], [SecurityProjects]
tags: [tag1, tag2, tag3]
---
```

### Available Categories
1. `TryHackMe` - TryHackMe challenges
2. `HackTheBox` - Hack The Box machines
3. `NetworkLabs` - Network configurations
4. `SecurityProjects` - Security tools/projects

### Recommended Post Structure
1. Overview (difficulty, platform, focus)
2. Enumeration (reconnaissance, scanning)
3. Initial Access (exploitation)
4. Post-Exploitation / Privilege Escalation
5. Key Takeaways

- Changes pushed to `main` branch trigger the GitHub Actions workflow

- **Source:** GitHub Actions (`pages.yml` workflow)


## Troubleshooting Reference

### Build Failures
- Check GitHub Actions logs
- Verify `_config.yml` syntax
- Check for invalid front matter

### Content Not Showing
- Verify filename format: `YYYY-MM-DD-title.md`
- Check date isn't in the future
- Validate YAML front matter

### Theme Issues
- Verify `theme: jekyll-theme-chirpy` in `_config.yml`
- Verify `gem "jekyll-theme-chirpy"` in `Gemfile`
- Check GitHub Actions build logs
